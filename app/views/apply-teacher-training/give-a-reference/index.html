{% extends "layout.html" %} {% set title = 'Give a reference' %} {% block pageTitle %}{{ title }}{% endblock %} {% block breadcrumbs %}{{ designHistory.breadcrumbs(breadcrumbItems()) }}{% endblock %} {% block content %} {% set content %} One of the first aspects of [the MVP service](/apply-teacher-training/apply-launch-26-nov-2019) we wanted to improve was how we accept references. Initially this was achieved using a Google form. However, this approach required a lot of manual intervention, preventing automation of reminders and status updates. Also, Google Forms are not very accessible. While maintaing the reduced question set, we devised a flow that would enable us to collect a reference and have its content be submitted directly to the service. With the limitation of Google Forms removed, we could also give referees the opportunity to correct any errors, and ask them to give a reason for declining to provide a reference. {% endset %}
